Alerts Module

Event monitoring, notification flow, and alert classification

Alert Event Stream

The alerts module is responsible for processing detection outcomes generated by the YOLOv9 inference pipeline and converting them into actionable system events. When fire or human presence is detected, the event is classified, logged, and optionally forwarded to external notification channels such as email and SMS.

Alert Classification

Critical Alert
Generated when fire is detected in the frame. This event has the highest priority and can invoke real-time notification delivery.


Moderate Alert
Generated when a human is detected in the monitored area without a fire event. Used for occupancy awareness and restricted-zone monitoring.


Composite Alert
Generated when both fire and human classes are detected simultaneously. Indicates elevated operational risk.

Notification Pipeline

Detection Layer
Receives class predictions from the inference engine.


Decision Layer
Applies event rules based on fire count, human count, and alert thresholds.


Delivery Layer
Sends email and SMS notifications and persists the event to the logs database.

Technical Notes

Input Source: Real-time video frames captured from the connected camera.

Detection Engine: YOLOv9 model with fire and person classes.

Processing Strategy: Per-frame inference followed by event classification logic.

Persistence: Alert records stored with timestamp, event label, and response status.

Integration: Supports email notification and SMS escalation for critical events.
