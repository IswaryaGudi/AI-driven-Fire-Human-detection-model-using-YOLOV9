System Architecture Summary

The system is designed as a real-time computer vision monitoring pipeline. Video frames are captured through OpenCV, processed by the YOLOv9 detection model, and passed into a response layer that updates the dashboard, generates event logs, and triggers notification services when configured conditions are met.


Frontend: HTML, CSS, JavaScript dashboard interface

Backend: Flask application server

Vision Layer: YOLOv9 inference engine

Communication Layer: Email and SMS alert services

Monitoring Goal: Early fire detection with human presence awareness
